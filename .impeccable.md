# Impeccable Design Context — quant-agent Dashboard

## Design Context

### Users
**사용자**: 박진영 (1인, 트레이더/개발자 겸업)
**주 사용 맥락**: 사후 분석·리뷰 — 거래 이후 에이전트 결정 과정 확인, 신호 흐름 검토
**해결할 문제**: 복잡한 10-팩터 신호 데이터를 한눈에 파악, 에이전트 판단 근거 빠르게 검토

### Brand Personality
**3단어**: 정밀한 · 신뢰할 수 있는 · 조용한
**감성 목표**: 불안이 아닌 통제감 — 데이터가 많아도 차분하게 읽히는 화면
**안티**: 화려한 효과, 넘치는 색상, 주의 분산 요소

### Aesthetic Direction
**방향**: 깔끔한 모던 대시보드
- 카드 중심 레이아웃, 큰 숫자, 충분한 여백
- 현재 다크 글라스모피즘 스타일 유지
- 핵심 지표(종합 점수, 가격, PnL)를 **계층 최상위**에 배치
- 차트: 큰 공간 확보, 캔들 가독성 우선
- 에이전트 로그: 타임라인 형태로 읽기 쉽게

**레퍼런스 레이아웃**:
```
┌──────────────┐  ┌──────────────────────┐
│  Price/PnL   │  │   CANDLESTICK CHART  │
│  큰 숫자      │  │   (넓게)              │
└──────────────┘  └──────────────────────┘
┌────────┐ ┌────────┐ ┌────────┐ ┌──────┐
│ Score  │ │  RSI   │ │  F&G   │ │ Pos  │
│  74    │ │  62    │ │  71    │ │+1.2% │
└────────┘ └────────┘ └────────┘ └──────┘
```

### Design Principles

1. **계층 명확화**: 종합 점수 > 가격/PnL > 개별 신호 > 로그 순서로 시각 무게 배분
2. **차트 우선**: 캔들스틱 차트에 최대 공간 할당 (사후 분석이 주 목적)
3. **정보 축소**: 현재 과다한 카드 수를 줄이고 핵심만 노출, 부가 정보는 접기/탭으로
4. **에이전트 로그 가독성**: BUY/SELL/HOLD 배지 + 이유 텍스트를 타임라인으로 명확히
5. **조용한 색상**: 강조는 teal(이익)·red(손실)·btc-orange(점수) 3색으로만, 나머지는 무채색
